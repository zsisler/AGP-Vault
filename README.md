# Obsidian Notes Repository

This repository contains my personal and professional notes maintained using **Obsidian**.  
The notes are written in Markdown and organized by topic and cadence (daily, weekly, study, etc.).

This repo serves as:
- A central knowledge base
- A version-controlled archive of notes
- A source of content for documentation, showcases, and knowledge sharing

---

## 📁 Folder Structure

### 🗓️ Daily Standup Notes
Notes used for daily standups, including:
- What I (or the team) worked on
- What I (or the team) plan to work on
- Blockers or follow-ups

Typically short, time-based entries.

---

### 📄 Procedures
Step-by-step documentation for:
- Technical processes
- Workflows
- Repeated tasks
- Setup or configuration guides

These notes are intended to be reusable references.

---

### 🎤 Showcases
Notes prepared for:
- Demos
- Presentations
- Knowledge-sharing sessions
- Internal or external showcases

Often more polished or structured than daily notes.

---

### 📚 Study Sesh Topics
Learning-focused notes, including:
- New technologies
- Concepts I’m studying
- Experiments or research
- Practice notes

These may evolve over time as understanding deepens.

---

### 🗓️ Weekly Meetings
Notes from recurring or scheduled meetings, including:
- Agendas
- Discussion points
- Action items
- Decisions

Used for tracking progress and historical context.

---

## 🛠️ Tooling

- **Editor:** Obsidian
- **Format:** Markdown (`.md`)
- **Sync:** GitHub (this repository)

---

## 🔄 Workflow

1. Notes are written and maintained in Obsidian
2. Changes are committed and pushed to GitHub
3. This repository acts as the source of truth
4. Others may add or pull at their convenience
